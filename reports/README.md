# Shared Expenses Management Platform

A robust, full-stack application designed to streamline group expense sharing, simplify net outstanding balances, track settlements, and automatically detect anomalies during bulk CSV imports. The platform is architected for speed, responsiveness, and exact mathematical accuracy.

## Overview

The Shared Expenses Management Platform (SplitWell) allows users to form groups (e.g., trips, apartments, events), add members, log expenses with customizable split types, and visualize who owes what to whom. Additionally, the platform features a powerful CSV import engine equipped with heuristic anomaly detection rules to catch common billing errors, name variations, and invalid transaction records before they are committed.

## Core Features

- **User & Friend Management**: Quick-registration of members with Name, Email, and Phone Number (default password autogenerated).
- **Group Management**: Group creation, user search, membership invite, and member removal.
- **Shared Expense Tracking**: Log expenses with description, total cost, split currency, and payer.
- **Greedy Debt Simplification**: Dynamic calculation of group balances and resolution of debts into the minimum number of payment transactions.
- **CSV Bulk Import**: Upload CSV logs of transactions.
- **Heuristic Anomaly Detection**: Automatic validation of imports checking for name variations, duplicates, negative amounts, incorrect currency logs, and membership timing conflicts.
- **Settlement Logging**: Real database logging of settlements between members to reduce outstanding balances.
- **Interactive Swagger Documentation**: Auto-generated interactive API docs for testing endpoints.

## Architecture

The platform follows a clean, decoupled three-tier architecture:

```
[ Frontend (React + Vite + Tailwind CSS) ]
                   │
                   ▼ (HTTPS Rest API / Axios)
   [ Backend (FastAPI + SQLAlchemy) ]
                   │
                   ▼ (PostgreSQL Connection String)
 [ Database (Neon Serverless PostgreSQL) ]
```

1. **Presentation Layer**: A single-page React app bundled with Vite, styled with Tailwind CSS, utilizing TanStack Router for route tree layout mapping, and Lucide icons for aesthetics.
2. **Application Layer**: A FastAPI Python service providing REST endpoints, using Pydantic for request/response serialization, and SQLAlchemy ORM.
3. **Data Layer**: Neon Serverless PostgreSQL instance storing normalized records of users, groups, memberships, expenses, anomalies, settlements, and import reports.

## Tech Stack

- **Frontend**: React (v19), Vite (v8), Tailwind CSS, TanStack Router, Lucide Icons, Fetch API/Axios.
- **Backend**: Python (v3.10), FastAPI, SQLAlchemy ORM, Pydantic (v2), Uvicorn.
- **Database**: Neon Serverless PostgreSQL.
- **Deployment**:
  - Frontend: Deployed on Vercel.
  - Backend: Deployed on Render.

## API Overview

Interactive documentation is available at `https://shared-expense-api.onrender.com/docs`.

### Authentication & Users
- `POST /auth/register` - Registers a new user.
- `GET /auth/users` - Fetches all registered users.
- `DELETE /auth/users/{user_id}` - Deletes a user, clearing associated memberships and expenses.

### Groups & Memberships
- `GET /groups/` - Lists all groups.
- `POST /groups/` - Creates a new group.
- `DELETE /groups/{group_id}` - Deletes a group and its memberships/expenses.
- `POST /memberships/` - Adds a user to a group.
- `GET /memberships/group/{group_id}` - Fetches memberships of a group.
- `DELETE /memberships/group/{group_id}/user/{user_id}` - Removes a user from a group.

### Expenses & Settlements
- `GET /expenses/` - Lists all expenses.
- `POST /expenses/` - Creates a new group expense.
- `DELETE /expenses/{expense_id}` - Deletes an expense.
- `GET /settlements/` - Lists all settlement payments.
- `POST /settlements/` - Records a settlement payment between group members.

### Analytics & Imports
- `GET /balances/` - Retrieves calculated user balances.
- `POST /import/` - Uploads a CSV expense log and runs anomaly detection heuristics.

## Setup Instructions

### Backend Setup
1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Create and activate a Python virtual environment:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Create a `.env` file with your connection string:
   ```env
   DATABASE_URL=postgresql://neondb_owner:npg_gmlExSL45pZO@ep-holy-feather-ahj6u3la-pooler.c-3.us-east-1.aws.neon.tech/neondb?sslmode=require
   ```
5. Run the FastAPI development server:
   ```bash
   uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   ```

### Frontend Setup
1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Run the Vite development server:
   ```bash
   npm run dev
   ```
4. Open your browser to `http://localhost:8081`.

## Deployment Information

### Frontend on Vercel
The frontend is deployed to Vercel and is configured to trigger automatic builds upon pushing changes to the `main` branch.
- Deployment URL: `https://mysplit-cyan.vercel.app`
- Configuration details are specified in `vite.config.ts`.

### Backend on Render
The Python service is deployed on Render running as a web service.
- API Endpoint URL: `https://shared-expense-api.onrender.com`
- Database tables are auto-migrated using SQLAlchemy `Base.metadata.create_all` during service startup inside `main.py`.
